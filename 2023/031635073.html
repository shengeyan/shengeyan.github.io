前端框架
发布日期:   2023-03-16

前端框架简介

解决什么问题

  • 前端要做的工作：根据用户的操作将相应的数据展示到视图中。
  • 为保证状态与UI同步，需要将大量的精力放在操作DOM上。
  • 频繁的操作DOM，这样的结果导致了代码臃肿，不易维护，容易出错，且浏览器渲染开销大。
  • 为了解决UI与状态同步的问题，出现了前端框架。
  • 前端框架帮助我们减少DOM的操作，同时提高了渲染页面的性能。

DOM操作

  • 访问操作dom比较耗时
  • 修改dom引起重排重绘，产生浪费

PS：

  • 重排：如果页面元素位置发生变化，就要从布局阶段重新开始渲染，也就是页面重排。页面重排一定会进行后续重绘。
  • 重绘：如果页面元素只是显示样式改变而布局不变，那么页面内容将会从绘制阶段开始。

渲染引擎的工作流程

解析html→构建DOM树→构建渲染树→渲染树布局→绘制渲染树

PS：

​ 渲染树：根据 可视化节点 和 css 样式表 结合诞生出来的树

Vue

简介

  • 用于构建用户界面的渐进式JS框架
  • Vue可以自底向上逐层应用

核心概念

虚拟DOM和DIFF算法

MVVM模型

M：模型（model）：对应data数据

V：视图（view）：模板

VM：视图模型：Vue实例对象

data bindings：数据绑定给view

dom listeners：监听 回馈view的数据改变

MVC模型

组件化

​ Vue将组成一个页面的HTML，CSS和JS合并到一个组件中，可以被其他组件或页面引入而重复利用。

​ 组件化很好的将一个庞大复杂的前端工程拆分为一个个组件，重复利用的性质也大大提高了开发的效率。（一个.vue就是一个组件）

响应式

声明式编码

命令式：侧重于实现

声明式：侧重于表现

生命周期

React

简介

  1. 一个JS UI库
  2. react一般作为MVC中的V（view）层，不依赖于其他库，可以与其他库的集成使用（Jquery，Backone）
  3. 在浏览器端运行，或通过nodejs在服务端渲染
  4. react语法为jsx，即在react中直接混合使用js和html，因此jsx运行需要将代码编译为普通js（可用多种思路解决 服务端通过webpack）

特点

  1. 声明式编程
  2. 函数式编程
  3. 组件方式开发
  4. 特殊语法JSX
  5. 虚拟DOM

函数式编程

函数类似于数据，也可保存，检索，传递

​ 核心概念：

  • 不可变性(Immutability)
  • 纯函数(Pure Functions)
  • 数据转换(Data Transformations)
  • 高阶函数 (Higher-Order Functions)
  • 递归
  • 组合

JSX

  1. JSX 是一种 JS 扩展的表达式

  2. JSX 是带有逻辑的标记语法，有别于 HTML 模版

  3. 并且支持样式、逻辑表达式和事件

  4. 类似于XML的JS拓展语法：JS+XML
