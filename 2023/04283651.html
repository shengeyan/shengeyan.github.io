Websocket&WebRTC

前端
发布日期:   2023-04-28

Websocket

应用场景

  • 服务器主动推送客服端
  • 双端通信
  • 即时通讯

其他即时通讯的方法

  • 定期轮询
  • SSE 服务端推送事件（单向）
  • comet技术(长链接)

如何创建websocket

  1. TCP三次握手创建连接
  2. 默认用http先进行一次通信
  3. 使用http请求进行升级，创建websocket连接

Webosocket的特点

  • WebSocket中所有发送的数据使用帧的形式发送。客户端发送的数据帧都要经过掩码处理，服务端发送的所有数据帧都不能经过掩码处理。
  • WebSocket创建必须是一个直接连接
  • WebSocket连接建立之后，通信双方都可以在任何时刻向另一方发送数据；
  • WebSocket连接建立之后，数据的传输使用帧来传递，不再需要Request消息；
  • WebSocket的数据帧有序。

相比于HTTP协议

  • http协议是半双工，websocket是全双工
  • http协议是从客服端主动发送请求，接受服务端响应
  • websocket可以从服务端主动推送信息给客服端
  • http实现伪websocket采取轮询方式

相比于socket

  1. socket基于操作系统
  2. socket是传输层接口

Websocket的优缺点

优点：

  • 跨域通信（CORS）—— 它会产生一定的安全风险，但这在复杂功能的项目中是必须的；
    跨平台兼容性（Web，电脑和移动设备）
  • 在需要服务端的 Web 应用程序或移动应用程序中，能够保持后端到前端的持续连接
    短时间的断连不会切断连接
  • 提供了完成异步任务的可能，而不局限于传统 Web 上的请求响应式任务

缺点：

  • 处理拒绝
  • 可能的隐私数据泄露
  • 客户端-服务端加密等

WebRTC

WebRTC是什么

​ 基于浏览器进行的实时通信技术

应用场景

  • 私人会议系统
  • 直播
  • 视频通信，实时通信…

WebRTC的优缺点

优点：

  • 无需额外安装软件；
  • 使用现代音频（Opus）和视频编解码器（VP8，H.264）进行高质量数据传输；
  • 视频或音频质量能在连接过程中根据具体环境进行调整，能抑制回声和噪音；
  • 数据的可靠性和安全性 —— 服务器通过 TLS 和 SRTP 协议进行数据加密以保证连接的安全和可靠；
  • 提供基于 HTML5 和 JavaScript 的无限制的控制接口；
  • 通过 WebSockets 可以使用多种后端技术来实现控制接口；
  • 开源软件的特点允许您将 WebRTC 嵌入 Web 应用程序和服务。
  • 浏览器成熟的API
  • 毫秒级的延迟特性

缺点：

  • 需要一个视频会议服务器（混合视频和声音）用于音频和视频会议，因为浏览器无法同步两个或多个输入信号流；
  • WebRTC 解决方案之间缺乏兼容性，不能调用其他开发者的 WebRTC 应用；
  • 您必须购买订阅服务器才能进行音视频的混合的会议。

WebRTC架构

点对点

​ 两个浏览器直接传送媒体内容

​ 优点：开发简单，成本低，安全性高

多点会议单元MCUS

​ 多个用户发送数据流。架构对每一个流进行解码，重新调整它们，从所有接收到的流中组合一个新的流，对其进行编码，然后将其发送回用户。

选择性转发单元SFU

​ 用户将媒体流发送到集中式服务器（SFU），并通过同一服务器接收来自其他用户的流。选择性转发单元 WebRTC 架构适用于 ADSL、移动和有线网络。

​ 优点：可扩展性高

WebRTC会话流程以及信令服务器搭建

核心对象：PeerConnection

作用：通过它完成呼叫端与被呼叫端关联，含有多个众多方法进行驱动工作

流程：

  1. A呼叫B，用websocket通信
  2. B应答，A,B均初始化自己的PeerConnection
  3. A调用createOffer创建信令，同时通过setLocalDescription方法在本地实例PeerConnection中存一份
  4. 调用信令服务器将 A 的SDP转发给 B
  5. B 接收到 A 的SDP后调用setRemoteDescription，将其储存在初始化好的PeerConnection实例中
  6. B 同时调用createAnswer创建应答SDP，并调用setLocalDescription储存在自己本地PeerConnection实例中
  7. B 继续将自己创建的应答SDP通过服务器转发给 A
  8. A 调用setRemoteDescription将 B 的SDP储存在本地PeerConnection实例

PS：

​ ice candidate：这个信息就是 ice 候选信息，A 发给 B 的 B 储存，B 发给 A 的 A 储存，直至候选完成。

WebRTC与Websocket的区别

  • WebSockets 的开发环境为 Java，JMS 和 C++；而 WebRTC 则是 Java 和 HTML
  • WebRTC 更加安全
  • 目前 WebRTC 只被部分浏览器支持，而 WebSockets 几乎与目前所有的浏览器兼容
  • 在可扩展性方面，WebSockets 的每个会话都会有一个对应的服务端，而 WebRTC 则是点对点式的
  • WebRTC 专为高性能媒体传输而设计。 WebRTC Web 应用程序通过服务或传输网络和媒体数据的网络运行

基础：前后端分离

  4. 模块化开发
  5. 组件化开发
