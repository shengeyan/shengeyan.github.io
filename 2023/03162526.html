React 官方中文文档 – 用于构建用户界面的 JavaScript 库 (docschina.org)

简介

  1. 一个JS UI库
  2. react一般作为MVC中的V（view）层，不依赖于其他库，可以与其他库的集成使用（Jquery，Backone）
  3. 在浏览器端运行，或通过nodejs在服务端渲染
  4. react语法为jsx，即在react中直接混合使用js和html，因此jsx运行需要将代码编译为普通js（可用多种思路解决 服务端通过webpack）

特点

  1. 声明式编程
  2. 函数式编程
  3. 组件方式开发
  4. 特殊语法JSX
  5. 虚拟DOM

函数式编程

函数类似于数据，也可保存，检索，传递

​ 核心概念：

  • 不可变性(Immutability)
  • 纯函数(Pure Functions)
  • 数据转换(Data Transformations)
  • 高阶函数 (Higher-Order Functions)
  • 递归
  • 组合

JSX

  1. JSX 是一种 JS 扩展的表达式

  2. JSX 是带有逻辑的标记语法，有别于 HTML 模版

  3. 并且支持样式、逻辑表达式和事件

  4. 类似于XML的JS拓展语法：JS+XML

XML

​ 早期用于存储与传输数据

虚拟DOM

优缺点

优点

  • 速度快，性能好（虚拟DOM）
  • 跨服务器兼容（虚拟DOM）
  • 单项数据流

缺点

  • 不是一个单独完整的框架

模块与组件、模块化和组件化

模块 模块化

​ 解释：原JS文件的不断扩大，将原JS分解为不同功能的JS文件，复用js，简化js编写，提高js运行效率

​ 模块化

​ 好处：

​ 1.方便维护，代码定位准确

​ 2.内部的变量，相互不影响，方便单个模块功能调试、升级，方便模块间组合、分解

​ 3.方便开发，多人协作互不干扰

​ 4.模块可以复用 

 缺点：

​ 1.系统分层，调用链会很长

​ 2.模块间发送消息会很耗性能

组件 组件化

​ 解释：实现某一功能的所有资源组合打包，替换类似ctrl+c ctrl+v 操作

JSX语法规则

  • 定义虚拟DOM事，不用写引号
  • 标签中混入JS表达式时要用{}

  • 样式类名指定不用class 用 className

  • 内联样式用：

  • 只有一个跟标签

  • 标签必须闭合

  • 标签首字母：

    1. 若小写字母开头，则标签转为html同名标签，若html没有同名标签，则报错
    2. 若大写字母开头，react渲染对应组件，若组件没有定义则报错
